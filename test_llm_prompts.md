# 🧪 Тестовые промты для проверки Smart Catalog Mode

## 🎯 **Цель:** Проверить как LLM понимает когда использовать catalog mode и может ли она находить нужные версии.

---

## 📝 **Тест 1: Проверка базового понимания**

### Промт:
```
Посмотри что нового в проекте open-webui на GitHub: https://github.com/open-webui/open-webui/releases

Ожидаю: LLM использует стандартный режим (5 releases)
```

### Ожидаемое поведение:
- LLM вызывает: `{"url": "https://github.com/open-webui/open-webui/releases"}`
- Получает: 5 последних releases
- Может ответить на вопрос "что нового?" по последним релизам

---

## 🎯 **Тест 2: Проверка понимания catalog mode**

### Промт:
```
Мне нужно найти в каком релизе open-webui добавили конкретную фичу. Сначала посмотри КАТАЛОГ ВСЕХ релизов проекта: https://github.com/open-webui/open-webui/releases?mode=catalog

Ожидаю: LLM понимает ?mode=catalog и использует его
```

### Ожидаемое поведение:
- LLM вызывает: `{"url": "https://github.com/open-webui/open-webui/open-webui/releases?mode=catalog"}`
- Получает: каталог всех 100 releases
- Видит метаданные всех версий
- Может искать конкретную фичу по preview

---

## 🎯 **Тест 3: Проверка LLM самостоятельного выбора**

### Промт:
```
Проанализируй релизы проекта open-webui: https://github.com/open-webui/open-webui/releases

Если тебе нужно больше информации о конкретных версиях для ответа, сначала посмотри полный каталог: https://github.com/open-webui/open-webui/releases?mode=catalog

Ожидаю: LLM сама решит использовать catalog или стандартный режим
```

### Ожидаемое поведение:
- LLM должна сначала вызвать catalog mode
- Проанализировать все 100 releases
- Затем решить нужны ли детальные данные

---

## 🎯 **Тест 4: Поиск конкретной фичи (реальный кейс)**

### Промт:
```
Пользователь спрашивает: "В каком релизе open-webui добавили oikb (knowledge base sync tool)?"

Сначала посмотри КАТАЛОГ ВСЕХ релизов open-webui: https://github.com/open-webui/open-webui/releases?mode=catalog

Проанализируй preview в каждом release чтобы найти где упоминается "oikb" или "knowledge base"

Ожидаю: LLM найдет нужную версию в каталоге
```

### Ожидаемое поведение:
- LLM вызывает catalog mode
- Проходит все 100 releases
- Находит в Release #5: "Official knowledge base sync tool oikb"
- Отвечает: "oikb добавлен в v0.9.6"

---

## 🎯 **Тест 5: Многоэтапный диалог**

### Промт:
```
First question: "Что нового в open-webui?" (https://github.com/open-webui/open-webui/releases)

Second question: "А в каком релизе добавили redirect-based SSRF protection?"

Ожидаю: LLM понимает что для второго вопроса нужен catalog
```

### Ожидаемое поведение:
- **Первый запрос:** Стандартный режим (5 releases) достаточно
- **Второй запрос:** LLM понимает что нужен поиск по старым версиям → использует catalog

---

## 🎯 **Тест 6: Сравнение режимов**

### Промт:
```
Проанализируй двумя способами:

1. Стандартный режим (5 latest releases): https://github.com/open-webui/open-webui/releases

2. Полный каталог (100 releases): https://github.com/open-webui/open-webui/releases?mode=catalog

Сравни количество информации и токенов в каждом режиме.
```

### Ожидаемое поведение:
- LLM вызовет оба режима
- Сравнит результаты
- Понимает когда какой режим лучше использовать

---

## 📊 **Критерии успеха:**

✅ **LLM понимает описание tool** и использует разные режимы
✅ **LLM может выбрать catalog mode** когда нужно искать по старым версиям
✅ **LLM может анализировать preview** в catalog для поиска конкретных фич
✅ **LLM может многоэтапные диалоги** с разными режимами

❌ **Провал:** LLM всегда использует стандартный режим даже когда catalog был бы лучше

## 📈 **Ожидаемые результаты:**

### **Идеальный сценарий:**
- LLM понимает когда использовать catalog
- LLM может найти конкретные фичи в preview
- LLM адаптирует поведение под тип вопроса

### **Реалистичный сценарий:**
- LLM иногда использует catalog, иногда стандартный режим
- LLM может требовать прямого указания ?mode=catalog
- LLM обучится на примерах в prompts

### **Неудачный сценарий:**
- LLM всегда использует стандартный режим
- LLM игнорирует catalog даже когда его очевидно нужно использовать
- LLM не понимает разницу между режимами
